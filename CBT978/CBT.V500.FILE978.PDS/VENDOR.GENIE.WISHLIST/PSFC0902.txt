NSample PSF Startup Procedures
N-----------------------------
NPSF Startup Procedures for AFP Printers
N---------------------------------------
NBefore starting a PSF FSS, you must have a cataloged startup
Nprocedure in a system procedure library, such as SYS1.PROCLIB.
NThis procedure specifies PSF initialization parameters and
Nlibraries that contain system and installation resources. The
Nname of this procedure can be specified in the FSSDEF statement
Nof the JES initialization deck. If the name is omitted, JES3
Nsupplies a default, chosen for the defined printer.
N
NSample PSF Startup Procedures
N-----------------------------
NA sample startup procedure for three FSSs, each with a single printer
NFSA. PRT1 is a channel-attached printer. PRT2 is an SNA-attached
Nprinter. PRT3 is any supported TCP/IP-attached printer, including any
Nprinter attached by use of the i-data IPDS Printer LAN attachment.
n

NJCL for PSF Startup Procedures
N------------------------------
NThis section describes the statements and the parameters.
N
NSyntax:
N-------
D//nnnnnnnn PROC
I  nnnnnnnn: Specifies the name of the startup procedure. The procedure
I            name must be specified in the JES2 FSS or the JES3 FSSDEF
I            initialization statement.
N
D//stepname EXEC PGM=APSPPIEP,
I  stepname: Specifies the name of the step.
N
I  PGM=APSPPIEP: Specifies the name of the PSF program.
N
D//             REGION=4096K,
I  REGION=4096K: Establishes the size of the region. How much memory
I                may be used.
N
D//             TIME=NOLIMIT,
I  TIME=NOLIMIT: Establishes the timeout limits. NOLIMIT will
I                not timeout.
N
D//             PARM=('INV=piname'|NSTddname,trace_type,prompt,
D//             trace_size,tcpip_name,UNICODE)
I  'INV=piname': Printer Inventory used: To use the Printer Inventory,
I                you must specify the Printer Inventory name as the
I                first and only parameter in the PARM field.
N
I                Specifies the four-character name of the Printer
I                Inventory that is specified in the Infoprint Server
I                configuration file. This parameter indicates that PSF
I                uses the Printer Inventory and obtains parameters from
I                the specified Printer Inventory for each printer in
I                the startup procedure. All other parameters in the
I                PARM field are ignored when PSF uses the Printer
I                Inventory.
N
I  NSTddname: Specifies the name of a DD statement defining the output
I             data set in which to record the NST trace output. This
I             name must conform to the standard JCL DD naming
I             conventions.
N
I             This parameter specifies that the NST trace is to start
I             during PSF initialization if you have specified TRACE=YES
I             on the PRINTDEV statement and have defined the NST trace
I             data set in a DD statement. If you want the NST trace to
I             be started dynamically, do not specify the NSTddname
I             parameter.
N
I  trace_type: Specifies the FSA trace to start:
I    FULL: Specifies an FSA full external trace.
I    SYNC: Specifies an FSA SYNC external trace.
I    INTR: Specifies an FSA internal trace and, if the printer is
I          SNA-attached, a PPCC internal trace.
I          INTR is the default value.
I    LIMIT: Specifies a shortened FSA external trace.
I    IPDS: Specifies an FSA IPDS external trace.
N
I    Notes: 1. For FULL, LIMIT, and IPDS, an FSA internal trace is also
I              started.
I           2. If you want to start the FSA trace dynamically, use the
I              default or specify INTR.
N
I  prompt: Specifies whether an operator response is required to
I          initialize the PSF operator interface:
I    PROMPT: Specifies that each time the FSS is initialized, the
I            operator is to receive a message, APS620A, that prompts
I            the operator to issue a response, which notifies PSF to
I            initialize the PSF operator interface. The response can be
I            any PSF operator interface command; it is directed to all
I            FSAs or to the notify subtask.
N
I            Thus the operator can type commands, such as those to start
I            FSI or FSA component traces, before PSF starts processing
I            data sets. Prompting occurs even if the startup procedure
I            does not include tracing specifications.
N
I    NOPROMPT: Specifies that the PSF operator interface is to be
I              initialized automatically. No operator response is
I              required.
I              NOPROMPT is the default value.
N
I  trace_size: Specifies the number of 4 KB pages of storage to
I              allocate for each internal FSA trace table. Valid values
I              are from 1 to 999. The default is 32 (128 KB). This
I              allocation occurs only if PSF internal tracing is active.
N
I    Note: When the number of pages specified is more than 32, and the
I          specified region is greater than 32 MB, increase the
I          PSF REGION size. To determine how large an increase is
I          needed, use the following equation where number of FSAs
I          active is the maximum number of FSAs active while the FSS is
I          running:
N
I   REGION increase = (number of FSAs active) x 4 KB x (pagecount - 32)
N
I  tcpip_name: Specifies the name of the TCP/IP address space. If this
I              parameter is not coded, PSF uses the default name of
I              TCPIP.
N
I  UNICODE: Specifies that PSF is enabled to use the system conversion
I           services that z/OS provides. If this parameter is omitted,
I           TrueType and OpenType fonts and QR Code bar codes with
I           SOSI data cannot be used.
N
I           When PSF provides Unicode-enabled support, Unicode encoding
I           transform interfaces, z/OS UNIX System Services C
I           interfaces, and file system interfaces are enabled.
N
D//outname OUTPUT CHARS=| ,
D//             COLORMAP=| ,
D//             COMSETUP=| ,
D//             COPIES| ,
D//             FORMDEF=| ,
D//             PAGEDEF=| ,
D//             PIMSG=| ,
D//             RESFMT=| ,
D//             TRC=
I  outname: Specifies a name for the statement.
N
I  OUTPUT: Specifies the page definitions, form definitions, and fonts
I          used to format and print different pages in a print job or
I          data set. In the examples, these pages include job and data
I          set header pages, job-trailer pages, and pages used for
I          printing messages.
N
D//ddname    DD ...
I  ddname: Specifies a name for the statement.
N
I  DD: Identifies the system libraries that contain resources
I      (fonts, page segments, overlays, page definitions, and form
I      definitions) used for printing. Also identifies the data set
I      where printer information is saved.
N
I      Note: If you are using Overlay Generation Language, any 3800
I            double-byte fonts must be in a library separate from other
I            double-byte fonts.
N
D//prtname CNTL
I  prtname: Specifies a name for the statement.
N
I  CNTL: Specifies the beginning of program control statements for the
I        printer. This statement must precede the PRINTDEV statement,
I        and has no parameters.
N
I        For JES2, the label on the CNTL, ENDCNTL, and PRINTDEV
I        statements must correspond to the printer name on the PRTnnnn
I        initialization statement.
N
I        For JES3, the label on the CNTL statement must correspond to
I        the label on the ENDCNTL and PRINTDEV statements and to the
I        name specified on the JNAME parameter on the JES3 DEVICE
I        initialization statement.
N
D//prtname PRINTDEV parameter
I  prtname: Specifies a name for the statement.
N
I  PRINTDEV: Specifies printer-initialization parameters. A PRINTDEV
I            statement is entered for each printer FSA belonging to the
I            FSS that names this PSF startup procedure. The label on
I            the PRINTDEV statement must match the label on the JES2
I            PRTnnnn initialization statement, or the JNAME parameter
I            on the JES3 DEVICE initialization statement.
N
I  parameter: The PRINTDEV statement specifies parameters for printer
I             initialization. In deferred-printing mode, PRINTDEV is
I             among the JCL statements in the PSF startup procedure.
I             In direct-printing mode, PRINTDEV is among the JCL
I             statements associated with the application program.
N
I    Parameter    Description
I    ------------ -------------------------------------------------------
I    ACQINTV (*)  Specifies the acquire interval in seconds. See
I                 Acquire interval on page 66 for more information.
N
I                 Syntax: ACQINTV=nnnnn, where nnnnn is a value from
I                   0 to 86400.
N
I    APPLID (*)   (required for SNA-attached printers only) Specifies
I                 the name of the VTAM application-program node for a
I                 functional subsystem application (FSA) when PSF is
I                 printing in deferred-printing mode. See Applid on
I                 page 67 for more information.
N
I                 Syntax: APPLID=name
N
I    BUFNO (*)    Specifies a factor for allocating the fixed I/O buffer
I                 storage used for channel-attached printers. See
I                 Channel buffer count on page 68 for more information.
N
I                 Syntax: BUFNO=nnnnn,
I                   where nnnnn is a value from 1 to 10000.
N
I    CHARS (*)    Specifies the last four characters of up to four
I                 default coded font names, which are used to print a
I                 data set on the printer. See Character sets on
I                 page 68 for more information.
N
I                 Syntax: CHARS=(fontname1,...¨)
N
I    COLORMAP (*) Specifies the member name of the printer default
I                 color mapping table. See Color map on page 70 for
I                 more information.
N
I                 Syntax: COLORMAP=(membername)
N
I    COMPRESS (*) Specifies whether PSF should compress data (blanks)
I                 in line data sent to printers, including printers
I                 attached to Infoprint Manager for Windows, or
I                 Infoprint Manager for AIX. See Blank compression on
I                 page 67 for more information.
M
I                 Syntax: COMPRESS=YES|NO
N
I    COMSETUP (*) (required for microfilm device only) Specifies the
I                 member name of the printer default microfilm setup
I                 resource object container generated with the utility
I                 provided with your microfilm device. See Com setup
I                 member on page 70 for more information.
N
I                 Syntax: COMSETUP=membername
