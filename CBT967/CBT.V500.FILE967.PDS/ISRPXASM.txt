* Instructions:
*
*   Change the CSECT for the language keyword tables you want to
*   alter.  To save space, You may delete CSECTs to delete entire
*   keyword lists.
*
*   If you want to change the symbols which are highlighted for a
*   language, modify CSECT ISRPXSYM.  You may specify up to 32 symbols
*   per language.  Do not change the length of the symbol entries or
*   delete any symbols entries.  Do not change the 1st entry.  It must
*   be blank.
*   Note that symbol changes are not reflected to users
*   until the user selects 'Defaults' on that language or 'Default
*   All Settings' from the HILITE dialog.
*
*   Assemble this file.
*   You MUST use the BATCH option (parameter) of the assembler.
*
*   Link the object as module ISRPX.
*   Note that the name of the load module is ISRPX, not ISRPXASM.
*   The module should be reentrant with RMODE(ANY),AMODE(31) and must
*   have the CSECT called ISRPX residing first in the load module.
*   Sample Linkage Editor Control statements are shown here:
*
*      ORDER ISRPX
*      ENTRY ISRPX
*       INCLUDE OBJECT(ISRPXASM)
*       MODE AMODE(31),RMODE(ANY)
*      NAME ISRPX(R)
*

* Optional instructions:
*   If you are going to place the modified copy of module ISRPX into
*   LPA, you may want to perform the following additional steps.
*   Although these steps are not required, they will result in
*   slightly better performance when the list is used:
*
*    1) Follow all of the steps above to create module ISRPX.
*    2) Place the new module in the ISPLLIB concatenation
*       and exit and reenter ISPF.
*    3) From an ISPF command line, type TSO ISRPLKWD.
*       This will create an assembler source file called
*       userid.ISRPXNEW.ASM.
*    4) Assemble ISRPXNEW.ASM
*    5) Re-linkedit the new object using the same control statements
*       shown above to create a new ISRPX load module.

*
*     Language       Table (CSECT) used
*     ------------   ---------------------
*     PL/I           ISRPXPLI
*     Pascal         ISRPXPSC
*     Cobol          ISRPXCOB
*     BookMaster     ISRPXSCR
*     REXX           ISRPXREX
*     Other          ISRPXGEN
*     C language     ISRPXC
*     JCL            ISRPXJCL
*     DTL            ISRPXDTL
*     IDL            ISRPXIDL
*     PL/X-370 (*)   ISRPXPLX
*     HTML           ISRPXHTM
*     XML            ISRPXXML
*     DTL Logic      ISRPXLGD  (tags with an end-tag)
*     HTML Logic     ISRPXLGH  (tags with an end-tag)
*     Assembler      None
*     Default        None
*     Panel          None
*     Skel           None
*     SuperC         None
*
*     BookMaster: ISRPXSCR
*        The only entries in the table are BookMaster tags which have
*        a corresponding end (:e) tag.
*
*     DTL logic words: ISRPXLGD
*        These are the tags that require an end tag.
*
*     HTML logic words: ISRPXLGH
*        These are the tags that require an end tag.
*
* (*) PL/X-370: ISRPXPLX
*        At the time of this writing, PL/X-370 is primarily an IBM
*        internal language which has limited external availability.
*        You may delete the CSECT ISRPXPLX entirely if you do not
*        use PL/X-370.
*
* All keywords are
*    DC AL1(keyword length + 1)
*    DC CLxx'keyword '  Must have a trailing blank
*
* All keyword lists end with
*    DC AL1(0)
*
* ******************************************************************
* ******************************************************************
* **                                                              **
* **  WARNING: Be sure that the length of the specified in the    **
* **           length field AL1(xx) matches the length            **
* **           specified for the keyword CLxx'...'.               **
* **                                                              **
* **           If the lengths do not match, the results are       **
* **           unpredictable.  This problem may show up as        **
* **           infinite loops or abends in the editor.            **
* **                                                              **
* ******************************************************************
* ******************************************************************
*
* Keyword lists must be sorted alphabetically.
*
* Each list is its own CSECT so lists for languages which are not used
* by your installation may be deleted.
*
* In CSECT ISRPXJCL, do not remove the word 'DLM '.  It is not
* highlighted, but is used to process delimited inline data
* statements.
*
*
